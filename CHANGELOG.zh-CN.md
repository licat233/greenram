# 更新日志

[English](CHANGELOG.md)

这里记录 GreenRAM 的重要发布变更。

## v0.13.0 - 2026-07-17

- 自动清理改为请求符合条件的 App 正常退出，不再立即强制结束进程。
- 将 macOS 内存压力事件接入普通 App 的清理 gate，并保留现有 RAM、Swap 和单 App 内存限制。
- 增加结构化清理决策，让菜单可以说明 App 是因为自动退出规则、系统内存还是自身内存上限而符合条件。
- 只有 macOS 接受退出请求后，才开始计算同一 Bundle ID 的重复请求冷却时间。
- App 内更新改为检查正式的 `licat233/greenram` 发布仓库。

## v0.12.1 - 2026-06-29

- 调整 legacy `.icns` 图标资源，为 macOS 13-15 增加透明边距和圆角，避免图标视觉偏大。

## v0.12.0 - 2026-06-20

- 将自动退出 App 和单 App 自动退出阈值分开；旧版单 App 时间规则会一次性迁移进自动退出 App，以保持升级后的原有行为。
- 将 App Rules 改成按规则类型进入二级页，并加入单 App 内存上限作为普通 App 的清理 gate。
- 阻止白名单 App 加入自动退出、单 App 超时时间或单 App 内存上限；必须先从白名单移除。

## v0.11.1 - 2026-06-13

- App 内自动更新继续优先使用已签名、已 notarize 的 `.app.zip` 发布资产。
- 当 release 没有 zip 资产时，自动更新可回退到安装 DMG。
- 自动更新安装时，如果 `spctl` 遇到临时系统错误，会回退使用 Gatekeeper 分发检查。
- Settings 中显示当前 GreenRAM 版本。
- 禁止将 GreenRAM 自己加入清理规则，且清理逻辑永不处理 GreenRAM 自己。

## v0.11.0 - 2026-06-13

- 加入基于 GitHub Releases 的 App 内一键更新，使用已签名、已 notarize 的 App zip 资产。
- 加入自动下载、解压、替换 App 并重启的更新安装流程。
- 更新安装前会校验 Bundle ID、版本、代码签名、Team ID 和 Gatekeeper 评估结果。
- 自动更新检查改为 GreenRAM 持续运行时也每天检查一次；只要仍有新版本，就会继续提醒安装。
- 自动更新优先选择 `GreenRAM-*.app.zip` 发布资产，没有可安装 zip 时回退到 DMG / 手动下载。

## v0.1.10 - 2026-06-13

- 重构 Settings 为更清晰的 SwiftUI 和 AppleViewModel 结构，并将 App 规则管理集中到独立页面。
- 重新设计菜单内存面板，复用 Settings 的环形指标，自适应高度，并简化 RAM 和 Swap 标签。
- 将 Swap 默认限制改为 8 GB，最高可配置 64 GB，并迁移旧版默认值。
- 启动时强制清除旧版 RAM 限制配置，统一回到 100%。
- 简化清理说明和菜单文案，包括“全部 App”和更清楚的自动退出 App 解释。

## v0.1.9 - 2026-06-08

- 加入基于 GitHub Releases 的更新检查，并提供菜单里的手动检查入口。
- 加入自动更新提醒，默认开启，并且每个新版本最多自动提醒一次。
- 在 Settings 中加入自动更新提醒开关。
- 通过系统打开优先发布资产，有 DMG 时优先下载 DMG。

## v0.1.8 - 2026-06-08

- 更新清理策略：自动退出 App 只等待自己的非前台时间；普通非白名单 App 还需要 RAM 或 Swap 状态超限；白名单 App 永不退出。
- 更新 Settings 和 README 文案，使其匹配新的清理策略。
- 删除过期的 MVP 说明文档。

## v0.1.7 - 2026-06-08

- 将可配置后台时间的最低值设为 3 分钟。
- 将超时清理改为使用明确的自动退出 App 列表。
- 让自动退出 App 列表和白名单互斥。
- 将内存阈值 UI 文案改为状态显示上限文案。
- 明确自动清理不会等待 RAM 或 Swap 超过限制。
- 澄清 Settings 和 README 中的自动退出 App 文案：列表中的 App 一旦达到非前台时间限制就会退出。

## v0.1.6 - 2026-06-08

- 发布支持 Apple Silicon (`arm64`) 和 Intel (`x86_64`) Mac 的 Universal 2 包。
- 在 Settings 中加入单 App 后台时间覆盖设置。
- 加入按 Bundle ID 配置后台时间阈值的策略和设置存储支持。
- 更新重置行为，重置时移除单 App 后台时间覆盖设置。
- 加入 README 兼容性说明和此更新日志。

## v0.1.5 - 2026-06-08

- 在 Settings 的白名单条目中加入 App bundle 选择。
- 改进白名单行，展示 App 名称、图标和 Bundle ID 详情。
- 为白名单展示缓存选中的 App 路径，并在移除条目时删除缓存路径。

## v0.1.4 - 2026-06-08

- 在 Settings 中直接加入白名单管理。
- 默认系统白名单条目改为可编辑，而不是永久保护。
- 重复退出冷却追踪从 PID 改为 Bundle ID。

## v0.1.3 - 2026-06-08

- 将清理策略改为使用非前台时长，而不是内存大小，作为 App 级清理条件。
- 加入可配置后台时间阈值，默认 30 分钟。
- 从清理决策中移除 App 类型、Bundle ID 关键词、App 名称关键词和风险分类检查。
- 保留 RAM 和 Swap 作为状态 / 阈值展示信号，不再作为 App 级清理 gate。

## v0.1.2 - 2026-06-08

- 更新菜单文案，区分可清理和不可清理 App。
- 将 Settings 截图加入项目文档。
- 优化清理候选相关的本地化文案。

## v0.1.1 - 2026-06-08

- 将 App 版本提升到 0.1.1。

## v0.1.0 - 2026-06-05

- 首个带 tag 的 GreenRAM 发布版本。
- 加入菜单栏内存状态、Settings、白名单支持、事件日志和本地化 UI。
- 加入 App 进程树的多进程内存统计。
